# 项目规则

## 模块简洁原则（核心）

**每个模块必须保持精简，禁止臃肿。**

### 每次工作前必须问自己：
1. 能更简洁吗？
2. 是否太臃肿？
3. 如果臃肿，怎么调整架构？

### 务实原则

**不要过度工程化。** 拆分前考虑：
- 项目是否有构建工具？（没有 → 慎重拆分）
- 数据文件 vs 逻辑文件？（数据文件长度可接受）
- 收益是否大于成本？

### 具体标准

| 维度 | 限制 | 说明 |
|------|------|------|
| 函数行数 | < 50 行 | 超过 80 行必须提取 |
| 参数数量 | ≤ 4 个 | 超过 5 个用对象封装 |
| 嵌套层级 | ≤ 3 层 | 超过 4 层必须重构 |

### 验证清单

- [ ] 单个函数是否做了多件事？
- [ ] 是否有重复代码可以提取？
- [ ] 新增代码是否让模块更臃肿？
- [ ] 新功能是否需要独立文件？
